Using ZML in your own projects

In this short guide, we will do the following:

We will turn the first_model into a C libary and call it from a C program.

The steps are:

  • start with a new folder for our project
  • copy over essential files from ZML's example folder
  • add Zig code to implement our model, based on the first_model
  • prepare for building with ZML:
    • add some lines of Bazel to integrate our code with ZML
    • and to build a static library
  • add a small C program that consumes our library
  • modify the BUILD.bazel to build and run our C program

Please note: the initial preparation steps are likely to get more streamlined.

If you haven't done so already, please install bazel:

macOS 
    brew install bazelisk
Linux 
curl -L -o /usr/local/bin/bazel 'https://github.com/bazelbuild/bazelisk/releases/download/v1.18.0/bazelisk-linux-amd64'
chmod +x /usr/local/bin/bazel

Setup

We copy over ZML's example folder, rename it to my_project for our project, and clean it up a bit.

# clone ZML if you haven't already
git clone https://github.com/zml/zml.git


# copy the ZML example folder
cp -pr zml/examples my_project

# erase unnecessary examples
rm -fr my_project/examples/llama
rm -fr my_project/examples/loader
rm -fr my_project/examples/mnist
rm -fr my_project/examples/simple_layer
rm -fr my_project/examples/third_party

We add a folder for our model, with the usual suspects main.zig and BUILD.bazel:

cd my_project
mkdir my_model
touch my_model/main.zig
touch my_model/BUILD.bazel
